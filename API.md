# Wi-Fi страница

Запрос HTML-страницы: `/wifi`

**Ко всем запросам, содержащим json необходимо добавлять соответствующий заголовок: `Content-Type: application/json`**

## Продолжение работы без подключения к сети

Отправить пустой POST-запрос на адрес `/wifi/without-connection`. Ответом придет запрос на redirect с HTTP-кодом 302 и
адресом на страницу выбора рецепта в заголовке на которую нужно перейти.

## Поиск доступных сетей

#### Этот запрос и все последующие возвращают `{"status":"ok"}` если действие выполнено, или `{"error": "Error message"}` при ошибке

Запрос доступных Wi-Fi сетей: GET-запрос на `/wifi/scan`

Возвращаемое значение: JSON-массив с найденными сетями, например:
```
{
  "networks": [
    {"ssid": "MyWiFi", "rssi": -65, "auth": "WPA2"},
    {"ssid": "Neighbor", "rssi": -80, "auth": "WPA/WPA2"},
    {"ssid": "OpenNet", "rssi": -70, "auth": "OPEN"}
    {"ssid": "", "rssi": -73, "auth": "WPA/WPA2"}
  ]
}
```
**Если сеть скрыта, то найденный ssid будет с пустой строкой!**
Скрытые лучше вообще не выводить просто оставить поля для ручного ввода где-то ниже

Показатель rssi показывает насколько сильный сигнал, оно всегда меньше нуля, но чем больше тем лучше сигнал, неплохо
будет список доступных сетей при отображении по нему отсортировать.

## Подключение к сохраненной сети 

### К любой из сохраненных доступных в данный момент:
Послать POST-запрос на `/wifi/connect`

### К определенной сети:
Послать POST-запрос на `/wifi/connect` в виде JSON-объекта:

```
{
    "ssid": имя_сети
}
```

## Другие endpoint-ы

Использовал [эту](https://github.com/tuanpmt/esp_wifi_manager/tree/master?tab=readme-ov-file) библиотеку, все оттуда 
актуальны, кроме `/vars`, может красивостей добавить вроде запроса статуса и вытащить оттуда quality и показывать 
уровень сигнала где-то в углу.


# Работа с рецептами и ингредиентами

## Ингредиенты

Адрес: `/ingredients`

### Получение списка ингредиентов

Отправить GET-запрос, пример ответа:
```
[
  {
    "id": 1,
    "name": "Водичка"
  },
  {
    "id": 2,
    "name": "Огненная водичка\uD83D\uDD25"
  },
  {
    "id": 3,
    "name": "Чай улун"
  }
]
```
В случае ошибки код 500.

### Обновление списка ингредиентов

Отправить PUT-запрос с новым списком ингредиентов.

При удалении нужно проверить, что:
1. Удаляемый ингредиент не используется ни в одном рецепте;
2. Удаляемый ингредиент не назначен на любой из насосов.

При добавлении нового:
1. Что такого ингредиента еще не существует;
2. Присвоить первый не занятый id (нумерация начинается с 1)

При простом переименовании никаких проверок не нужно.

Структура json-а такая же, как в ответе с сервера.

Ответом придет код 201 Created в случае успеха, или 500 в случае ошибки.


## Рецепты

Адрес: `/recipes`

### Получение списка рецептов

Отправить GET-запрос, пример ответа:
```
[
  {
    "id": 1,
    "name": "Кровавая Мэри",
    "ingredients": [
      { "ingredientId": 1, "amount": 50 },
      { "ingredientId": 3, "amount": 100 }
    ]
  },
  {
     "id": 2,
    "name": "Джин-Тоник",
    "ingredients": [
      { "ingredientId": 4, "amount": 50 },
      { "ingredientId": 5, "amount": 150 }
    ]
  },
  {
    "id": 3,
    "name": "Текила Санрайз",
    "ingredients": [
      { "ingredientId": 6, "amount": 50 },
      { "ingredientId": 2, "amount": 120 },
      { "ingredientId": 7, "amount": 20 }
    ]
  }
]
```
В случае ошибки код 500.

### Обновление списка рецептов

Отправить PUT-запрос с новым списком рецептов.

При удалении нужно проверить, что:
1. Удаляемый рецепт не выбран в данный момент (как вариант если выбран то сразу послать запрос на выбор другого);
2. Сдвинуть id всех рецепты после удаленного, так чтобы все id шли подряд.

При добавлении нового:
1. Что такого рецепта еще не существует.

При простом переименовании никаких проверок не нужно.

Структура json-а такая же, как в ответе с сервера.

Ответом придет код 201 Created в случае успеха, или 500 в случае ошибки.

# Системные команды

## Обновление прошивки
На адрес `/firmware-update` отправить POST-запросом .bin файл с новой прошивкой, ответом придет 500 при ошибке, либо 200, если
прошивка загружена. После загрузки прошивки контроллер перезапустится.

## Перезапуск контроллера
На адрес `/restart` отправить POST-запрос, ответом придет 500 при ошибке, либо 200, если все ок, после чего
контроллер перезапустится.
